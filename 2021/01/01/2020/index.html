2020 年终总结

Jan 1, 2021
文章目录
  1. 1. 工作内容
  2. 2. 参与社区
  3. 3. 未来已来 2021

时间飞快，还来不及多写几篇博客，2020已经结束。这时，总不免感叹时间都去哪儿了！

年初疫情导致全国各地的人都在家自我隔离，开启远程办公模式。没想到年末疫情有所反复，我仍然在家办公。在家办公对互联网行业/工作的人影响不大，甚至是利好。

对我个人而言，省去了上下班在路上的时间，能多睡会，在家穿着打理上更加随意，办公效率方面基本不受影响。工作方面，由于各种原因，9月份换了新的公司，工作内容的核心也发生了变化，不再直接参与消息中间件相关的研发。两年的基础研发对个人而言还是很受益的，非常感谢原团队的TL、同事的关照。对基础研发方面的爱好转变为参与相关社区的建设，期待未来在开源社区有更多的贡献。

工作内容

工作内容的变化主体现在从基础架构相关的研发转到业务的研发，因此结识了一群新的同事、朋友，和业务贴的更紧密，有了更多行业的视角。

基础研发还是业务？

我觉得处于工作的不同阶段都可以尝试，没有好坏，适合自己就行。如果你看好一个行业，当然要去最前线，将个人的发展和行业的发展结合起来，撸起袖子好好干，期待几年后有个好结果。

参与社区

开源社区及项目有很多，目前只是尝试参与了一些与日常工作相关的开源项目，更多的是尝试，还没有找到可以长期持续参与的项目。目前参与了Apache ShardingSphere/Bookkeeper/Pulsar、K8s等开源社区，有少量的贡献，期待在接下来的一年有更多的参与。

参与开源的方式除贡献代码外，文档、翻译都是不错的方式，这一年基本搞清楚参与开源的那些方法(套路)。简单列举下参与的一些PRs：

  • [翻译]全职工作者如何为 Kubernetes 做贡献
  • [字幕翻译]为Apache纪录片提供字幕翻译
  • [Apache Pulsar] Redirect Get message by id request when broker not serve for the topic
  • 更多我的Github 2020年度

未来已来 2021

新的一年，技术的发展，感觉云原生(K8s、微服务)、响应式编程是趋势，顺势而为，未来可期。

工作方面，期待疫情尽早结束，公司、部门的业务发展更上一层楼，个人作出更多的成就。

该努力的年龄，就要不留遗憾，坚持价值投资，做时间的朋友！
